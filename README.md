## Подход к разработке
1. Не лезьте в чужой код (не редактировать самостоятельно)
2. Предупреждайте над чем будете работать чтобы не было ситуаций в которых 2 человека пишут одно и то же
3. После выполнения задачи помечайте ее как выполненую
4. Одна задача - один коммит (Не объединяйте в одном комите 1000 разных изменений)
5. Старайтесь делать все в отдельных, специализированых файлах
6. Нововведения в уже существующих сценах делайте на копиях этих сцен(У меня были ситуации когда такое ломало ВЕСЬ проект)
